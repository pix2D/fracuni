# Puppeteer for PDF generation

Invoice PDFs are generated by rendering HTML templates through headless Chromium via Puppeteer. This lets us reuse the same HTML/CSS used for browser preview as the PDF source — one template, two outputs. The Docker image uses a Puppeteer-ready base image with Chromium pre-installed. The Chromium dependency is heavy (~200MB), but on a Docker volume on an internal machine this is irrelevant.

**Considered:** `@react-pdf/renderer` (limited layout primitives), `pdfkit` (manual positioning), Typst (less mainstream). All would require maintaining a separate template system from the web preview.
